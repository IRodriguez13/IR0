## Release gate vs audit

| Gate | Incluye audit |
|------|----------------|
| `make smoke-release-0.0.1` | `linux-abi-audit` (todos los enabled) |
| `make linux-abi-audit` | **16** contratos enabled |
| `make agent-fast` | host + ktest (~20s) |
| `make diag-contract CONTRACT=…` | ISO fresco + un contrato |

**FAT16 rw:** no habilitar hasta `linux-abi-audit-vfs-write` verde **sobre backend FAT16**.

Fuente ABI dup CLOEXEC: [man 2 dup](https://man7.org/linux/man-pages/man2/dup.2.html).

**Deuda ARCH (no stub):** `proc_open` / `sysfs_open` aún devuelven fds virtuales (`FD_PROC_BASE`+). `dup`/`fcntl`/`dup2` operan sobre `fd_table` real. Unificar pseudo-fs opens en slots de `fd_table` (como `devfs`/`pseudo_bind_dir_fd`) es el siguiente sprint de lifecycle, no un éxito fingido sobre 1500+.
