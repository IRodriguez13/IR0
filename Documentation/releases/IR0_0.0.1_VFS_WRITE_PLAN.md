# IR0 0.0.1 — VFS write-path plan (post-stat freeze)

> **Last verified:** 2026-06-26  
> **Source of truth:** `scripts/linux_abi/contracts.json`, `Documentation/releases/IR0_0.0.1_ABI_BOARD.md`, `fs/vfs.c`

## Meta

Antes de **FAT16 write** o nuevos FS mutables, congelar contratos VFS de escritura auditados contra Linux con el mismo ELF (`scripts/linux_abi/workloads/*_probe.c`).

Backends objetivo (orden):

1. **MINIX rw** — primario en `disk.img` / rootfs tier-1  
2. **FAT16 rw** — secundario (smoke read-only ya en gate D1.18)  
3. **EXT2 / IR0FS / UNIXFS** — post-0.0.1  

**Regla:** no implementar semántica de escritura dentro de FAT16 (ni otros backends) hasta que el contrato VFS correspondiente tenga audit PASS.

## Cadena de contratos (orden obligatorio)

| # | Contrato | Gate Makefile | Bloquea |

| 1 | stat / fstat | `linux-abi-audit-stat` | metadata en write paths |

Cada iteración: Linux workload → IR0 workload → compare → primera divergencia → fix mínimo → commit solo en PASS.

## stat/fstat (puente actual)

Workload: `stat_probe.c` — `stat("/proc/uptime")`, `stat("/sbin/init")`, `stat("/sbin/noent")`, `open`+`fstat`, `fstat(-1)`.

Compare (estable cross-OS):

- ret / errno  
- `st_mode & S_IFMT` (tipo)  
- `st_size > 0` cuando aplique (no igualdad cross-OS salvo archivo fijo)  
- `st_nlink` informativo si difiere  
- **No** comparar: inode, timestamps exactos  

## open-creat (siguiente slice)

Casos mínimos Linux:

- `openat(O_CREAT|O_EXCL)` → EEXIST  
- `openat(O_CREAT|O_TRUNC|O_WRONLY)` truncate existing  
- `openat(O_APPEND)` posición inicial al final  

Backend: tmpfs o MINIX en path bajo directorio existente (`/sbin/` o tmpfs mount).

## write roundtrip

- Crear archivo pequeño, `write` patrón hex, `read` verify, `close`  
- Comparar bytes + errno en short write / EPIPE no aplica (archivo regular)

## Implementación backend (después de audits)

| Fase | FS | Acción |
|------|-----|--------|
| W1 | MINIX | `ops->create`, `write`, `truncate` ya parciales — alinear errno/flags tras audits |
| W2 | FAT16 | rw solo tras contratos 1–3 verdes en MINIX; reutilizar VFS generic semantics |
| W3+ | EXT2/IR0FS | diseño separado post-0.0.1 |

## Anti-patterns

- FAT16 write “para probar” sin audit open/write  
- Flags Linux (`O_APPEND`, `O_TRUNC`) interpretados solo en backend  
- Cambiar layout `struct stat` userspace sin host `stat_user_abi`
